# 09_結合テスト仕様書_Ver1.0
# 1. 文書概要

## 1. 目的
本書はAI運用保守プラットフォームの結合テスト仕様を定義する。Power Apps、Power Automate、AI Builder、Microsoft Lists、SharePoint、Teams、Outlook、SLA、権限制御、通知、AIおよびエラー処理を対象とする。

## 2. 適用範囲
- Power Apps
- Power Automate
- AI Builder
- Microsoft Lists
- SharePoint Online
- Outlook
- Microsoft Teams
- SLA管理
- 権限制御
- 通知
- エラー処理

## 3. 関連設計書
01～08の正式設計書を前提とする。

## 4. 方針
単体試験完了後に実施し、コンポーネント間連携を重点確認する。AI失敗時も業務継続を確認する。


---


<!-- 09_01_結合テスト方針_Ver1.0.md -->


# 2. 結合テスト方針

- ブラックボックス試験を基本とする。
- 実運用に近いシナリオで確認する。
- CorrelationIdによりPower Apps～Power Automate～SystemErrorまで追跡する。
- 正常系・異常系・権限制御・性能観点を含める。

## 合格基準
- 想定結果と一致
- データ整合性維持
- エラー時も設計どおり継続または停止
