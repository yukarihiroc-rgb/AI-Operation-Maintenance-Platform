# Power Apps 時刻入力方式比較

## 概要

Power Apps には DatePicker のような標準の時刻ピッカーはありません。
そのため、時刻入力は以下のいずれかの方法で実装するのが一般的です。

-   ① ドロップダウン（推奨）
-   ② コンボボックス（Outlook風）

------------------------------------------------------------------------

# ① ドロップダウン（推奨）

## 構成

-   DatePicker
-   時（Hour）Dropdown
-   分（Minute）Dropdown

### イメージ

``` text
┌──────────────┐
│ 2026/07/20 📅 │
└──────────────┘

┌────┐ ┌────┐
│14▼│ │30▼│
└────┘ └────┘
```

## メリット

-   Microsoft Learnでも一般的な構成
-   実装が簡単
-   Power Apps標準コントロールのみで構成可能
-   保守性が高い
-   入力ミスが発生しにくい

## デメリット

-   コントロール数が増える
-   Outlookほど見た目はスマートではない

## おすすめ用途

-   業務アプリ
-   入力画面
-   運用保守システム

------------------------------------------------------------------------

# ② コンボボックス（Outlook風）

## 構成

DatePicker と時刻一覧の ComboBox を組み合わせます。

### イメージ

``` text
┌──────────────┐
│ 2026/07/20 📅 │
└──────────────┘

┌──────────────┐
│14:30 ▼       │
└──────────────┘
```

候補例

``` text
09:00
09:15
09:30
09:45
10:00
・・・
23:45
```

## メリット

-   Outlookに近い操作性
-   画面がすっきりする
-   利用者が直感的に操作できる
-   15分・30分単位などの制御が容易

## デメリット

-   時刻リストを作成する必要がある
-   1分単位にすると候補数が多くなる

## おすすめ用途

-   Outlook風UI
-   スケジュール入力
-   社内業務アプリ

------------------------------------------------------------------------

# 比較

  項目            ① ドロップダウン   ② コンボボックス
  --------------- ------------------ ------------------
  実装難易度      ★★★★★（簡単）      ★★★★☆
  保守性          ★★★★★              ★★★★☆
  見た目          ★★★☆☆              ★★★★★
  操作性          ★★★★☆              ★★★★★
  Outlookらしさ   ★★☆☆☆              ★★★★★

------------------------------------------------------------------------

# Phase1-Aへの推奨

AI運用保守プラットフォーム Phase1-A では、

**② コンボボックス（Outlook風）** を推奨します。

理由

-   Outlookに近い操作性
-   運用担当者が迷いにくい
-   画面がすっきりする
-   社内システムとの親和性が高い

一方で、実装のシンプルさを最優先する場合は、①ドロップダウン方式も十分に実用的です。
