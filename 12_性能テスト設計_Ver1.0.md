# 12_性能テスト設計_Ver1.0

# 1. 文書概要

## 1.1 目的

本書は、AI運用保守プラットフォームにおける性能テスト設計を定義することを目的とする。

Power Apps、Power Automate、AI Builder、Microsoft Lists、SharePoint Online、Outlook、Microsoft TeamsおよびSLA監視機能について、性能要件、試験方針、試験環境、試験方法、評価方法および合格基準を定義し、本番運用前に必要な性能品質を確認する。

本設計書は「08_テスト基本設計_Ver1.0」を補完し、性能試験実施時の標準設計書として位置付ける。

# 2. 性能テスト方針

## 2.1 目的
システムが要求性能を満たし、20名規模の利用において安定して運用できることを確認する。

## 2.2 基本方針
- 実運用を想定した性能試験を実施する。
- Microsoft 365標準サービスの利用制限を考慮する。
- ボトルネックを特定できる試験とする。
- AI性能と業務性能を分離して評価する。
- 性能低下時でも業務継続可能であることを確認する。

## 2.3 試験対象
- Outlookメール受付
- Power Apps
- Power Automate
- AI Builder
- Microsoft Lists
- SharePoint Online
- Teams通知
- SLA監視

## 2.4 試験種別

|試験|内容|
|---|---|
|応答時間|画面・処理速度|
|負荷試験|連続実行|
|同時利用|複数利用者|
|AI性能|解析時間|
|通知性能|Teams・メール|
|再試行性能|Retry動作|
|監視性能|SLA・ログ|

## 2.5 評価観点
- 応答時間
- スループット
- 安定性
- エラー率
- Retry成功率
- AI解析時間
- 通知遅延
- SLA監視時間

## 2.6 試験条件
- 約20名利用
- Microsoft Lists実データ
- 実運用構成
- Outlook実メール
- AI Builder利用

## 2.7 除外事項
- Microsoftデータセンター性能
- ブラウザ性能
- インターネット障害
- Microsoft内部サービス性能

## 2.8 合格方針
- 業務停止なし
- データ欠損なし
- Retry正常
- CorrelationId追跡可能
- 設計性能を満足

## 2.9 設計レビュー

|観点|結果|
|---|---|
|性能方針|適合|
|運用性|適合|
|保守性|適合|
