# 第1章 文書概要

## 1.1 目的

本書は、Phase1-A受付機能先行リリースにおける実装工程の進捗を管理することを目的とする。

実装計画書で定義された工程ごとの進捗状況を可視化し、各工程の着手状況、レビュー状況および完了状況を一元管理する。

## 1.2 適用範囲

本書はPhase1-A実装計画書で定義された以下の工程を対象とする。

-   Step1 SharePoint構築
-   Step2 受付関連Microsoft Lists作成
-   Step3 Power Apps（閲覧専用GUI）
-   Step4 GUIレビュー
-   Step5 AI Builder
-   Step6 Power Automate
-   Step7 結合テスト
-   Step8 受付機能先行リリース

## 1.3 情報源

本書は以下の正式成果物を情報源として管理する。

-   Phase1-A_受付機能実装計画書_Ver1.0
-   AI運用保守プラットフォーム設計書_Ver1.6
-   AI共同開発運用標準_Ver2.2
-   プロジェクト運用ルール_Ver2.4

# 第2章 管理方針

## 2.1 管理対象

管理対象は実装工程単位とする。

各工程について、

-   開始
-   実装
-   レビュー
-   完了

までの状況を管理する。

## 2.2 ステータス

  状態         説明
  ------------ --------------------------------
  未着手       作業開始前
  作業中       実装中
  レビュー中   レビュー実施中
  完了         レビュー完了・次工程へ移行可能
  保留         課題等により一時停止

# 第4章 管理ルール

## 4.1 更新タイミング

-   工程着手時
-   ステータス変更時
-   レビュー開始時
-   レビュー完了時
-   工程完了時

## 4.2 状態遷移

``` text
未着手
   │
   ▼
作業中
   │
   ▼
レビュー中
   │
   ├──差戻し────────┐
   ▼                │
完了 ◀─────────────┘

保留は任意の状態から設定可能
```

## 4.3 管理上の注意事項

-   実装順序は実装計画書に従う。
-   レビュー完了後に次工程へ進む。
-   保留となった場合は理由を備考欄へ記載する。
-   実装課題、レビュー指摘事項および改善提案は「Phase1-A_実装管理台帳」にて管理する。
